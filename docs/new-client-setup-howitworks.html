Setting up a new client

What the admin does · what gets invoked · what gets stored — org-agent orchestration

Mobius org-setup · verified live 2026-07-17

what the user does services invoked data stored
1

Create the client

UserIn /admin/clients, type the org name + state → “create”.
Invokes chat proxy /api/v1/org/setup org-agent POST /org/setup org-intelligence (discovery) credentialing /org/upsert
Stores org_profile · chat DB slug · NPIs · locations · aliases
2

Provision the knowledge base

automatic — part of create
UserNothing — sees a “Setting up…” pill turn to “Ready”.
Invokes org-agent /datastore/provision credentialing /doc-store/provision db-agent provisioner
Stores per-org schema · mobius_org_docs org_documents · org_chunks (pgvector) doc_store descriptor · chat DB
3

Onboard employees

UserAdd people (name + roles), one or many → “invite”.
Invokes org-agent /employees/invite mobius-user /users/invite mobius-email (set-password link)
Stores app_user (invited) · mobius_user user_org_membership (roles) auth_token (invite)
4

Employee activates

UserEmployee clicks the email link → /auth/set-password → sets a password.
Invokes chat /api/v1/auth/token-info mobius-user /auth/set-password
Stores password_hash · status → active session · mobius_user
5

Dashboard (ongoing)

UserOpens the org page — a live progress hub: Identity · Knowledge base · Team.
Invokes org-agent GET /org/{slug} credentialing + mobius-user (aggregate)
Stores nothing — read-only aggregation
The org-agent stores nothing. It is pure orchestration — it sequences the calls above and aggregates the reads. Every piece of data lives in the subsystem that owns it, so there is one source of truth per concern and no duplicated state.

Where the data lives

org_profileOrg identity — slug, NPIs, locations, aliases. Owned by credentialing (chat Postgres DB).
mobius_org_docsPer-org knowledge base — a Postgres schema per org with pgvector chunks. Provisioned by db-agent; ingested by RAG; role-isolated.
mobius_userAccounts, org memberships + roles, invite/reset tokens, sessions. Owned by mobius-user.
mobius-emailSends invite / set-password / reset emails; keeps a send audit.
